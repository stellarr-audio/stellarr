# AI Agent Development Guidelines

## Initial Setup

- Read and understand the existing codebase structure, dependencies, and patterns
- Provide a concise summary of the project's purpose, tech stack, and architecture
- Wait for explicit approval before making any code changes or additions

## Development Approach

- Follow existing code patterns, naming conventions, and project structure
- Ask for clarification when requirements are ambiguous rather than making assumptions
- Suggest solutions but wait for confirmation before implementing
- Prioritise simple, maintainable solutions over complex ones
- Respect existing testing patterns and coverage levels

## Communication Standards

- Keep responses focused and actionable
- No emojis in any output, logs, or generated content
- No auto-generated documentation files unless requested
- No conversation summaries or meta-files
- Ask before creating additional tests beyond what's requested
- Do not summarise the conversation after each successful code edit

## Writing Style (All Content)

- Use New Zealand English spelling and conventions
- Use present tense verbs instead of past tense
- Write factual statements and direct commands, avoiding hypotheticals
- Use active voice where the subject performs the action

## Workflow Requirements

- Explain your approach and ask for permission to proceed before implementing significant changes
- Highlight any potential breaking changes or dependencies
- Suggest improvements to existing code only when directly relevant
- Focus on the immediate task rather than broad refactoring
- Confirm understanding of guidelines before beginning work

# Documentation Standards

- Generate documentation only when explicitly requested
- Write clear and concise documentation with consistent terminology and style
- Write in second person to speak directly to readers
- Use headings to organise content and bullet points for lists
- Include links to related resources when relevant
- Do not include project file structure as it's redundant
- Do not include code examples in documentation
